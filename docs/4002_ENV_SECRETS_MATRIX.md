# Env & Secrets Matrix

目的：定義環境變數與秘密資訊的配置矩陣。
避免憑證散落、權限過大與環境不一致。

## 環境分層
- `local`：開發者本機測試。
- `deploy-preview`：PR 預覽環境。
- `production`：正式環境。

## 變數清單矩陣
- 逐項列出：變數名稱、用途、是否敏感、適用環境、管理位置。
- 敏感變數不得提交至 repo。
- 非敏感公開設定可放在可版本控設定檔。

## 秘密管理與輪替
- 敏感值僅放 Netlify/GitHub Secrets。
- 建立定期輪替與離職即撤銷機制。
- 輪替時需驗證 deploy-preview 與 production 正常。

## 權限與稽核
- 以最小權限提供 Secret 讀取與修改權。
- 變更需可追溯（人員、時間、原因）。
- 重大變更需雙人複核。

## 事故處理
- 發現洩漏立即撤銷與替換憑證。
- 評估影響範圍並啟動回滾或緊急修補。
- 事件後更新安全文件與 runbook。
