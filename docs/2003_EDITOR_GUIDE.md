# Editor Guide

目的：提供編輯人員可執行的寫作規範。
降低審稿與發布過程中的格式錯誤。

## 標題層級規則（H1/H2/H3）
- H1 由 frontmatter `title` 產生，內文不手寫 H1。
- 文章主章節使用 H2，子章節使用 H3。
- 不跳階使用標題（避免 H2 直接跳 H4）。
- 每個 H2 段落應有清楚主題與至少一段正文。

## 圖片 alt/caption 規則
- 所有內容圖片使用 Cloudinary URL，不上傳到 repo。
- 每張圖片都要有 alt，描述畫面重點與用途。
- 需要補充上下文時，加上 caption 說明。
- 避免把純裝飾圖用於文章核心資訊。

## tags/日期/dateModified 規則
- `tags` 使用小寫英文與連字號，避免同義重複標籤。
- `pubDate` 為首次發布日，不因小修正變動。
- 內容更新時同步調整 `dateModified`。
- 草稿請保持 `draft: true`，審核通過再改 `false`。

## 寫作與格式建議
- 段落先結論後細節，單段建議不超過 4-5 行。
- 連結文字使用可理解語意，避免「點此」。
- 程式碼區塊標註語言類型，便於閱讀與高亮。
- 發布前自檢：標題層級、連結、拼字、metadata 完整性。
